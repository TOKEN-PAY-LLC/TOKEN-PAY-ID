Условия использования

Последнее обновление: 13 марта 2025 г.

Настоящие Условия использования (далее — «Условия») регулируют отношения между ООО «ТОКЕН ПЭЙ» (ИНН 4706094495, ОГРН 1254700021267, далее — «Оператор») и физическим или юридическим лицом (далее — «Пользователь»), использующим сервисы, доступные по адресу tokenpay.space (далее — «Сервис»).

1. Общие положения

  1. Использование Сервиса означает полное и безоговорочное согласие Пользователя с настоящими Условиями.
  2. Оператор оставляет за собой право изменять Условия без предварительного уведомления. Актуальная версия доступна по адресу tokenpay.space/terms.html.
  3. Если Пользователь не согласен с Условиями, он обязан прекратить использование Сервиса.

2. Описание Сервиса

Сервис TOKEN PAY ID предоставляет:

  • Систему регистрации и аутентификации пользователей
  • Личный кабинет для управления аккаунтом и API-ключами
  • Публичный API (REST) для интеграции со сторонними приложениями
  • OAuth 2.0 авторизацию для сторонних сервисов
  • SDK для встраивания кнопки входа через TOKEN PAY

3. Регистрация и аккаунт

  1. Для использования Сервиса необходима регистрация с указанием действительного адреса электронной почты.
  2. Пользователь обязуется предоставить достоверные данные при регистрации.
  3. Пользователь несёт ответственность за сохранность учётных данных (логин, пароль, API-ключи).
  4. Один Пользователь может иметь только один аккаунт.
  5. Оператор вправе заблокировать или удалить аккаунт при нарушении настоящих Условий.

4. API-ключи и доступ к API

  1. API-ключи являются конфиденциальной информацией. Пользователь обязан обеспечить их безопасное хранение.
  2. Использование API осуществляется в рамках установленных лимитов (rate limiting).
  3. Запрещается использование API для: рассылки спама, DDoS-атак, распространения вредоносного ПО, нарушения законодательства РФ.
  4. Оператор вправе отозвать API-ключ при обнаружении злоупотреблений.

5. OAuth 2.0 и сторонние приложения

  1. Пользователь вправе авторизоваться в сторонних приложениях через TOKEN PAY ID (OAuth 2.0).
  2. При авторизации Пользователь явно подтверждает передачу запрашиваемых данных стороннему приложению.
  3. Оператор не несёт ответственности за действия сторонних приложений после передачи данных.
  4. Пользователь может в любой момент отозвать доступ стороннего приложения через личный кабинет.

6. Интеллектуальная собственность

  3. Использование товарных знаков TOKEN PAY и TOKEN PAY ID допускается только в порядке, предусмотренном документацией SDK.
